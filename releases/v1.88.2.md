# v1.88.2

## Added

- Implemented Bitget USDT-FUTURES Best 1m market-data support: REST downloader,
  manual job queue integration, PBData latest-refresh loop, Market Data UI/API
  dispatch, Heatmap/Jobs support, and Bitget config defaults.
- Added optional distributed Bitget Best 1m backfill over selected downloaders:
  the local worker splits each coin into small dynamic chunks, can use the
  master as a local downloader, runs a threaded stdlib-only remote Bitget REST
  downloader over SSH for VPS downloaders, streams raw candles back, and writes
  `.npz`/source-index files only on the master.
- Added live distributed Bitget downloader traffic details in Jobs: expandable
  per-downloader cards show payload size, rows, pages, segments, written
  minutes, and status for each VPS/Master downloader.
- Added a Jobs Details log fallback for older distributed Bitget jobs so the
  expandable downloader traffic cards can reconstruct VPS contributions from
  existing `segment done` log lines when live per-downloader stats were not yet
  recorded in the job JSON.
- Kept the Jobs downloader traffic expander open across live WebSocket refreshes
  so the per-VPS contribution view no longer collapses immediately.
- Filled the active Jobs downloader traffic view from the live job log when the
  running worker has not yet written per-downloader progress into the job JSON,
  so VPS contribution cards are no longer shown as empty while work is ongoing.
- Parsed `payload=...` values from distributed Bitget job logs so log-derived
  downloader cards show transferred payload bytes instead of `0 B`.

## Fixed

- Started Bitget Best 1m jobs with a fresh one-shot task worker so stale
  resident workers cannot fail them as an unknown job type.
- Set Bitget Best 1m REST defaults below the official `history-candles` /
  `candles` endpoint limit (`18 req/s`, 16 workers) and added shared 429
  backoff across worker threads so temporary Bitget public rate limits slow the
  job down instead of failing it immediately.
- Fixed Jobs Monitor duration display for retried/requeued jobs by tracking the
  actual run start/finish timestamps and preferring recorded worker duration.
- Added visible pending and success feedback for service restart actions on the
  Services page.
- Fixed distributed Bitget VPS SSH targets to use the known VPS hostname/SSH
  alias instead of the raw IP, avoiding host-key verification failures.
- Stopped distributed Bitget jobs from logging the full embedded remote
  `python3 -c` downloader source; job logs now show a short downloader start
  message instead.
- Made distributed Bitget jobs resilient to transient SSH downloader failures:
  failed SSH segments are requeued, the failed downloader is retired, and any
  remaining requeued segments are completed by the master fallback instead of
  failing an otherwise complete job.
- Retried transient distributed Bitget SSH segment failures on the same VPS
  before retiring the downloader, added default SSH timeout/keepalive options,
  and increased the default dynamic chunk size to 30 days to reduce SSH startup
  overhead.
- Made distributed Bitget jobs resolve each coin's REST inception day when no
  explicit start date is provided, preventing pre-listing empty history chunks
  from being scheduled across VPS downloaders.
- Streamed distributed Bitget segment planning so VPS downloaders can start on
  each coin as soon as its inception day is found instead of waiting for every
  selected coin to be probed first.
- Made distributed Bitget planning skip local days that are already complete so
  VPS downloaders do not refetch existing `.npz` data when `refetch` is off.
- Changed Jobs downloader traffic cards to show only the currently active
  Bitget coin/range per downloader instead of the cumulative list of all coins
  a downloader has processed.
- Changed Jobs downloader traffic cards to display written Bitget data as days
  instead of raw minutes to avoid ambiguous thousands separators.
- Added mouseover explanations for Jobs downloader traffic metrics such as
  rows, pages, segments, payload, written days, and status.
- Fixed OKX latest-1m success logs being written as `ERROR` because the success
  payload contains an `errors: 0` field that triggered the generic log-level
  heuristic.
- Made distributed Bitget planning skip unavailable symbols reported by Bitget
  as `40034 Symbol not exists` so one delisted/missing coin no longer fails the
  entire backfill job.
- Fixed Bitget Best 1m queue normalization so mapping coins such as `HYUNDAI`
  are no longer shortened into invalid Bitget symbols such as `HYUN`, and
  unsupported selected coins are rejected before a job is queued.
- Added visible progress for Optimize `Add to Archive`, moved the archive export
  work off the API event loop, and made the PBv7 frontend report non-JSON server
  responses cleanly instead of showing a raw JSON parse error.
