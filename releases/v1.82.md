# v1.82

- Fixed VPS live telemetry handling so stale metrics streams are restarted, repeated stale streams reconnect SSH, stale CPU alerts are suppressed, and VPS Manager/Monitor mark old host telemetry as stale instead of online/live.
- Added a PB7 config-schema compatibility guard for v7 live saves/restores/syncs so PBGui warns in an explicit OK modal to update the target VPS before pushing a newer config schema, and removed the misplaced `max_warmup_minutes` field from the Backtest editor.
- Removed backdrop-click closing from shared and page-local dialogs so modal windows require an explicit close, cancel, or confirm action.
- Fixed Pareto Explorer full-result loading so `Load all` ranks configs from `all_results.bin` by the selected load strategy, honors PB7 `min`/`max` scoring goals, and recomputes the visible Pareto front instead of pinning saved `pareto/*.json` entries.
- Fixed Pareto Explorer Best Match scoring to use the selected chart metrics with normalized profit/risk directions, and corrected drawdown load strategy ranking for positive drawdown values.
- Fixed Pareto Explorer preview charts so the `Use weighted (_w) metrics` toggle prefers available weighted scoring metrics instead of falling back to unweighted labels, and Best Match scoring now considers all active preview/quick-view axes including 3D metrics.
- Reduced Pareto Explorer 3D scatter marker sizes so dense fronts remain readable.
- Fixed additional Pareto Explorer metric-direction inconsistencies in full-mode performance selection, Command Center summaries/champions, Deep Parameters near-bound ranking, config-detail scoring, full-config reconstruction, and correlation/radar selection.
