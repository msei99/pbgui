# v1.91

- Removed the tracked API server log containing token-bearing URLs and excluded it from Git.
- Fixed VPS Linux update counts remaining stale when the live package probe verified an outdated IP host key instead of the trusted inventory hostname.
- Fixed valid IPv4 CIDR firewall sources blocking VPS configuration saves during Linux update actions.
- Fixed PBData dropping buffered prices when a database batch write fails.
- Secured fresh PBGui installs with individual generated web passwords, VPN-only remote API firewall access, and a visible warning for legacy default credentials on wildcard listeners.
- Prevented stored and reflected DOM XSS in API-key user rows and both job monitors without changing their layout or controls.
- Moved unsaved Hyperliquid private keys from expiry-check query strings into authenticated POST request bodies.
- Replaced the mutable third-party Telegram commit action with a permissionless, repository-controlled notification step.
- Fixed Optimize V7 Save As renaming the opened config instead of creating an additional config.
- Restricted PBGui credentials, sessions, API-key backups, and Cluster secret blobs to owner-only files and directories, including automatic migration of existing installations.
- Closed active privileged WebSockets immediately on logout or session expiry.
- Moved browser session authentication to an HttpOnly SameSite cookie and removed session tokens from page, SSE, WebSocket, and Pareto Dash proxy URLs while retaining Bearer authentication for API clients.
- Redirected stale protected browser pages to login after session loss instead of displaying a raw authentication error.
- Fixed standalone Help and embedded Job Monitor pages still requiring a token URL after browser authentication moved to cookies.
- Kept Guide help on the current page through a lazy-loaded shared overlay and made the standalone Help fallback return to its referring PBGui page when closed.
- Serialized API-key, dashboard, task-queue, INI, AWS-profile, and market-data read-modify-write transactions across threads and processes to prevent lost updates and shared temporary-file collisions.
- Fixed PBRun low-memory recovery ranking V7 bots by an unrelated monitor payload instead of their attached process memory statistics.
- Preserved valid mapped Tiingo quote-cache entries when equity or FX refresh requests fail or return only partial results.
- Prevented stale Dashboard and Optimize Queue WebSocket callbacks and already-dequeued reconnect timers from reviving replaced connections.
- Prevented older Dashboard and Orders HTTP responses from overwriting newer filter, timeframe, or position views.
- Released stopped remote-log streams and unused REST/CCXT-Pro exchange clients instead of retaining them indefinitely.
- Added bounded per-client login throttling with escalating temporary lockouts and standards-compliant retry responses.
- Added authenticated Login Security status to the Welcome overview using active blocks and retained Auth log history.
- Added persistent acknowledgement for Welcome login-security alerts; newer lockouts automatically raise the warning again.
- Made intentional No-Login operation an explicit confirmed authentication mode with persistent global warnings, full session rotation, audit logging, and bounded per-client passwordless session reuse.
- Added graceful FastAPI shutdown tracking for DB Tools operations, sync jobs, and Cluster repair/join/push jobs without stopping independent Passivbot backtest or optimize processes.
- Moved scheduled and manual DB Sync execution into persisted detached one-shot workers with independent SSH connections, restart-safe progress, dead-worker recovery, and API restart blocking only for non-detachable DB Tools or Cluster operations.
- Completed API background lifecycle management: shared Live/Dashboard watchers, WebSockets, Backtest/Optimize controllers, Heatmap and archive streams, Pareto loaders and Dash sessions, Coin Data refreshes, VPS deployment/import controllers, and cached exchange clients now shut down deterministically, while persistent OHLCV and Passivbot jobs survive API restarts and reconcile from disk.
- Preserved HTTP 409 for blocked API restart requests instead of converting expected conflicts into HTTP 500 responses.
- Aligned the OpenAPI and English/German Help documentation with the actual message envelopes, parameters, routes, and cookie authentication of all active WebSocket endpoints.
- Added an explicit confirmation before permanently deleting V7 instance backups without interrupting routine actions with unnecessary prompts.
- Separated deterministic offline tests from explicitly enabled live Exchange, external PB7, and local-runtime tests while isolating generated market data from production directories.
- Made the Pytest configuration, shared fixtures, active tests, test dependencies, and portable test commands visible and reproducible instead of hiding them through local ignore rules.
- Documented the audit-derived authentication, secret handling, path validation, concurrency, stale-request, background-lifecycle, restart, and test-isolation invariants for future contributors.
- Removed the completed code-audit tracking roadmap after all actionable findings were addressed or intentionally deferred.
