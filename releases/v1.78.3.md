# v1.78.3

Released: 19-05-2026

- Fixed VPS Manager update button highlighting so `Update PBGui` and `Update PB7` pick up the same GitHub update warning state already shown in the version cards, including cases where the detail status flag lags behind the summary row.
- Fixed local VPS Manager master sudo actions such as `Update Linux` so cached master sudo passwords are reused only after a fast local validation check, the UI re-prompts automatically when the cached password is no longer valid, confirmed password prompts no longer drop the pending master action before it starts, and local Linux updates now use the same safe scheduled reboot path as `Reboot Master` instead of Ansible's `reboot` module on a localhost connection.
- Fixed local VPS Manager PBGui version reporting after `Update PBGui` so the master card reads the updated version directly from the checkout on disk instead of keeping the stale version constant from the still-running API process until restart.
- Removed `Cleanup Bucket` from the local master sidebar because bucket cleanup is a VPS-focused action and should only be offered in VPS context.
- Changed the Overview `Cleanup Bucket` button to use a cached non-blocking background indicator: it turns green when the bucket is clean, orange only when matching cleanup prefixes exist, runs once on first Overview entry, and is invalidated only by bucket-relevant actions such as setup, delete, or cleanup itself.
