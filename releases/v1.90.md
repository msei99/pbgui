# v1.90

- Blocked Add VPS initialization until required pre-flight checks, including `/etc/hosts` and SSH connectivity, are green.
- Added VPS Manager `Add to Cluster` support and automatic local Cluster node registration after successful VPS setup, without joining/remotely changing the VPS.
- Documented the exact workflow for joining a newly set up VPS to an existing Cluster, including master sync-peer activation and Login Key status meaning.
- Added a detailed implementation plan for a dedicated VPS monitor agent service that preserves current monitor behavior while avoiding duplicated expensive measurements across multiple masters.
- Added the first VPS monitor-agent implementation and wired VPS setup/update/migration flows to install and restart `pbgui-monitor-agent.service`; masters now read live metrics from the agent cache instead of launching duplicate inline collectors.
- Completed the monitor-agent cache flow for instance snapshots, host metadata, service status and package status; masters now read `data/monitor_agent` cache files instead of starting periodic remote collectors, and VPS Manager shows monitor-agent health/age details.
- Added monitor-agent audit fixes so systemd migration status requires `pbgui-monitor-agent.service` and bot log detail popups use log-tail reads instead of legacy collector scripts.
