# v1.78.1

Released: 18-05-2026

- Removed PBGui version/origin/branch-history parsing from `PBRun` and delegated those lookups to `pbgui_release.py` so runtime bot management no longer owns PBGui release metadata.
- Removed the remaining PB7 and Python-version metadata helpers from `PBRun`; VPS status views now read release/runtime interpreter metadata from dedicated providers or direct interpreters instead of the runtime bot manager.
- Removed local Linux upgrade/reboot probing from `PBRun`; the VPS Manager now reads local maintenance state directly inside `vps_manager_service.py` instead of via the bot runtime manager.
- Fixed the new PBGui/PB7 release providers and remote host-meta version probe so VPS Manager overview rows show real PBGui/PB7 versions and branch status again instead of `N/A`/`unknown`.
- Removed PBGui release metadata from `PBRemote`; the VPS Manager now keeps its own local release-status cache while `PBRemote` stays focused on remote config/sync concerns.
- Moved the full historical changelog out of `README.md` into dedicated files under `releases/` and reduced `README.md` to release-note links plus product documentation.
- Fixed VPS Manager release/status cards staying stale after Master PBGui update or branch-switch actions by forcing a fresh backend status reload.
- Fixed additional FastAPI pages with hash/query URL state so browser reload keeps auth parameters after in-page navigation.
- Fixed FastAPI VPS Manager browser reloads after in-page navigation so Master/VPS subviews keep the auth query parameters instead of dropping back to login.
- Fixed the VPS Manager PBGui branch dropdown so the branch selector stays open while choosing a target branch on Master and VPS branch views.
- Fixed VPS monitor SSH flapping alerts when only the remote metrics subprocess ended and the SSH connection itself was still alive.
- Fixed the top navigation VPSMonitor alert dialog so active alerts open correctly.
- Fixed VPSMonitor alert history so repeated host offline episodes are preserved instead of only showing the latest outage.
- Extended VPSMonitor alert history retention to keep resolved alerts for 1 week instead of only returning the latest 20 entries.
