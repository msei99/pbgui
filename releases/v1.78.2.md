# v1.78.2

Released: 19-05-2026

- Simplified VPS Manager ownership boundaries further: local PBGui/PB7 version, branch, Python, and package-maintenance status now come from dedicated release/runtime providers instead of being tunneled through `PBRun` or `PBRemote`.
- Fixed VPS Manager remote host metadata refresh so PBGui/PB7 version and branch fields update reliably again after VPS actions, including repaired SSH host-meta probing and removal of the stale legacy snapshot fallback from the active FastAPI monitor path.
- Added an explicit VPS Manager `Cleanup Bucket` workflow with lightweight preview, dry run, direct delete from the dry-run dialog, detailed delete-result logs, and clearer host classification for bucket entries.
- Hardened `Cleanup Bucket` safety checks and UX: root bucket directories returned by `rclone lsf --recursive` no longer cause false failures, the result dialog is no longer overwritten by follow-up modals, and failure details now stay visible in the dedicated result view.
- Improved VPS Manager task visibility: task logs now default to newest-first with a working sort toggle, real `--run-<id>` log filenames sort correctly, and Overview rows now show a compact `Task Running` indicator with the active command and current step.
- Hardened API restart behavior around VPS deploys by blocking restart only for genuinely active tasks, recovering stale deploy state on startup, ignoring leftover completed ansible processes, and surfacing restart blockers clearly in both the nav and Services monitor.
- Fixed `Cleanup VPS` so it uses the same staged VPS sudo-password flow as `Update Linux`, preventing cleanup runs from failing after the frontend correctly prompts for the password.
- Added a PBRemote one-time local cleanup for obsolete `data/cmd/status.json` and `data/cmd/status_single.json` artifacts on VPS hosts while leaving bucket cleanup as an explicit VPS Manager action.
- Fixed the v7 Run editor/API load path so `live.price_distance_threshold` is restored correctly from raw config data instead of falling back to the editor default when PB7 materializes it as `None`.
