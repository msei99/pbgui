# v1.78

Released: 17-05-2026

- Changed FastAPI `VPS Manager` into the main remote operations hub: the Overview now supports multi-host selection, `Bots` counts, click-to-sort headers, per-column hide/reset controls, compact PBGui/PB7/GitHub status indicators, and a dedicated shared `History`/deploy-progress view.
- Added shared FastAPI `VPS Manager` actions for `Deploy Settings`, `Update Linux`, `Update PBGui`, `Update PB7`, `Update PBGui and PB7`, and `Cleanup VPS`, including saved `parallel`/`sequential` rollout mode, `Debug`, `Reboot after Linux update`, host-by-host sudo validation for Linux updates, and per-run deploy logs.
- Added FastAPI `VPS Manager` `VPS Logging` controls and rollout support so one shared logging config for `PBRun`, `PBRemote`, `PBCoinData`, `sync`, `vps_cleanup`, and `tradfi_sync` can be saved in PBGui and deployed to selected VPS or automatically during `Setup VPS`.
- Added a dedicated remote `Update PB7` action and expanded `Cleanup VPS` so VPS management now covers PBGui, PB7, Linux updates, logging rollout, and broader runtime cleanup from the same FastAPI UI.
- Added historical monitoring overlays and persisted telemetry for VPS and bot metrics: host CPU/Memory/Disk/Swap history, bot CPU/Memory/Swap history, realized PNL and fills history, plus 4-week error/traceback history with restart-safe rebuilds from remote logs.
- Changed VPS and bot monitoring to use the shared FastAPI/API stack end-to-end, including persisted monitor snapshots, recent alert history, acknowledge actions, clearer threshold labels, and shared nav alert indicators instead of the older mixed PBRemote/PBMon paths.
- Improved VPS Manager live telemetry and detail refresh so quick/full detail hydration, websocket reconnects, task logs, host logs, deploy progress, package/reboot state, uptime, RTD, and bot tables stay current without flicker or manual reloads.
- Added richer VPS `Host Logs` and `Task Logs` workflows with archived bot logs, multi-window/floating log viewers, direct branch-task log opening, cleaner ANSI/Ansible rendering, and stable reopen/restart behavior for the selected live log.
- Fixed VPS Manager detail, delete, and import edge cases: deleted hosts now fall back cleanly to `Overview`, imported/master hosts handle missing live telemetry safely, explicit navigation targets are stable, and browser refreshes restore the expected detail view.
- Changed remote host handling so PBGui install paths, host metadata, service checks, log access, and Ansible playbooks now work across both `~/software/pbgui` and `~/pbgui`, including imported PBGui master servers managed via SSH.
- Hardened VPS credential handling: login/sudo/init secrets are session-only, no longer persisted in host JSON, temporary runner directories are cleaned up correctly, and related setup/detail flows were adjusted to work without writing secrets back to disk.
- Expanded `Add VPS` and setup validation with live `/etc/hosts`, SSH, bucket, CoinMarketCap API-key, firewall, and init-method checks, including guided `/etc/hosts` add/replace actions and safer password validation for Ansible/Jinja-sensitive input.
- Changed FastAPI/API-key sync handling so `API Sync` now uses the shared SSH worker status across `API Keys`, `VPS Manager`, and related FastAPI pages, with clearer out-of-sync reporting and quick-push actions for managed VPS.
- Added FastAPI `VPS Manager` master `Host Logs` support and improved master/VPS branch management with dedicated `PBGui Branch` and `PB7 Branch` views.
- Added the standalone FastAPI root Welcome/Login flow with a minimal login page, browser token bootstrap, password change, PB7 setup, and direct entry into the migrated FastAPI pages.
- Fixed standalone PBv7 editor state leaks across FastAPI `Run`, `Optimize`, and `Backtest`, so stale local multiselects, suite/chip state, limits, scoring, coins, and tags no longer bleed into other configs when switching, loading, importing, or saving under a different config name.
- Fixed PB7 optimize `Results` so pareto-only result directories now stay visible even when `write_all_results` does not produce `all_results.bin`, preventing completed runs with only `pareto/*.json` from disappearing from the web UI.
- Improved logging and housekeeping across the runtime stack: the Logging Monitor now supports capped single-file logs (`backup_count=0`) reliably, PBRemote/PBRun log writers respect the configured rotation settings, OHLCV preload jobs now log under `data/ohlcv_preload/logs/`, and stale preload artifacts are cleaned up automatically.
- Updated the English and German VPS Manager guides to match the current FastAPI UI, actions, and workflows.
- Added a dedicated Welcome/Login help topic and unified the Guide window behavior across the migrated FastAPI pages.
