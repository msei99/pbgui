# v1.88.1

- Added a Market Data Copy Data panel that queues safe rsync-based OHLCV copies to SSH targets with missing-only defaults, no remote delete, data-tip field label tooltips, guide examples, and a read-only connection/destination test.
- Added a Copy Data dry-run queue action that runs rsync with `--dry-run --stats --itemize-changes`, skips remote directory creation, and shows target/transfer details in the job log before any real copy.
- Fixed Copy Data dry-run jobs so they start through a fresh one-shot worker instead of being picked up by an already-running older task worker that does not know the new dry-run job type yet.
- Fixed real Copy Data jobs the same way: copy jobs now also start through a fresh one-shot worker so stale resident workers cannot fail them with `Unknown job type: ohlcv_copy`.
- Added an inline Copy Data dry-run summary that follows the queued job and shows the remote root, exchanges, files to transfer, transfer size, and rsync stats directly in the Copy Data panel.
- Fixed the Copy Data dry-run summary to aggregate rsync stats across all selected exchanges instead of showing only one exchange's stats block.
- Moved Copy Data dry-run transfer totals into structured worker progress so large multi-exchange dry runs no longer depend on truncated job logs for their summary.
- Reordered the Copy Data form so the SSH command field appears before the final remote target field.
- Fixed Market Data settings saves so changing enabled coins or auto-enable queues the exchange's latest-1m refresh immediately instead of waiting for the existing interval sleep.
- Reduced PBData startup-delay logging so shared poller delay status is logged once per delay window instead of every second, and empty user queues are no longer logged.
- Added an Optimize Config import dialog that accepts pasted JSON, a local `.json` file, or a Backtest Archive config, prepares it through the existing PB7 config normalizer, and opens it as an unsaved editor draft.
- Clarified agent instructions so API/startup edits always bump `api/serial.txt`, including later API edits in the same session.
