# v1.86

## Highlights

- Introduced Cluster Sync with the PBCluster daemon as the supported remote sync path for multi-master and VPS runner setups, replacing PBRemote, API Sync, V7 SSH Sync and API-key FileSync for V7 configs and `api-keys.json`.
- Added the Cluster Sync page and APIs for local identity, node membership, desired V7 state, tombstones, operation visibility, read-only remote probes, remote state previews and explicit sync/rebuild actions.
- Added role-aware materialization: masters receive all desired V7 configs, VPS runners receive only assigned configs, and API keys are materialized through verified payload/secret blobs with master-only replacement backups.
- PBRun now treats Cluster Sync desired state as the runtime gatekeeper before starting or continuing V7 bots, while PBCluster only replicates and materializes state.

## Cluster Operations

- Added bootstrap, remote join and second-master self-join flows with guarded recovery, automatic metadata detection, one-shot SSH password prompting, progress polling and dedicated restricted PBCluster SSH keys.
- Added editable node sync modes (`Disabled`, `Outbound Only`, `Reachable via SSH`), per-node outbound peer allowlists, Repair SSH, disabled-node removal and safer handling for unreachable or uninitialized peers.
- Cluster remote writes now use explicit preview/apply steps, bulk operation/blob upload where available, streamed payloads for large transfers, local progress tracking and fallback paths for older remotes.

## VPS Manager And Services

- Added Import Cluster Nodes to preview and import safe SSH metadata from Cluster Sync nodes into VPS Manager without copying passwords, sudo secrets, CoinMarketCap keys or private keys.
- Import Cluster Nodes can update local `/etc/hosts` through the normal sudo prompt, use per-host passwords only for the current session, install monitoring keys, reconnect the running monitor and show live job progress.
- PBCluster is included in local and remote service monitoring, systemd migration checks, branch-switch/update handling, host log viewing and local Services controls.
- Pure VPS runners no longer need `pbgui-api.service` or `PBApiServer.py`; master-only nodes can run PBApiServer/PBCluster without forcing PBRun.

## Migration And Documentation

- Added and refreshed English and German guides for Cluster Sync, Cluster Mode migration and VPS Manager workflows, including node modes, PBCluster restarts, self-join, runner/API boundaries and API-key materialization.
- Removed obsolete PBRemote surfaces and cleanup paths now remove stale PBRemote systemd units where found.

## Stability

- Hardened API startup so launches from `/` or `//` normalize to the PBGui checkout before routers/logging can use relative `data` paths.
- Improved Cluster Sync recovery for missing bootstrap-era blobs, parallel materialization races, stale config blocks, tombstone restores and backup restore naming.
