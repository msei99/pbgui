# v1.83

## Highlights

- Added a new master installer entry point (`setup/master_installer.sh`) that can launch a local browser wizard or CLI from one command.
- Added Local Master install and uninstall modes for installing PBGui/PB7 locally, handling prerequisites, and removing local user services with explicit confirmation.
- Improved Local Master uninstall path handling so existing systemd user units determine the default install parent and only units that belong to the selected parent are stopped or removed.
- Added systemd user-service setup for PBGui services so new master and VPS installs no longer rely on legacy `start.sh` and crontab autostart.
- Added Services and VPS Manager migration flows for converting existing master and VPS installs to systemd user services with dry-run previews, validation, logs, and safe legacy autostart cleanup.
- Added configurable PBGui install paths across VPS setup, updates, logs, V7 config sync, API-key backups, and service restarts instead of assuming `~/software`.
- Added an Existing VPS import flow that probes over SSH, previews detected paths/processes/crontab entries, and saves the VPS without running init or setup.
- Improved Existing VPS import process detection so systemd-managed PBGui services are not reported as legacy processes.
- Added automatic SSH key installation when saving an Existing VPS import so live monitoring can be enabled without manual `ssh-copy-id` steps.
- Improved remote master installation with fresh-VPS confirmation, SSH host-key fingerprint acceptance, branch selection, prerequisite handling, clearer password fields, reveal toggles, elapsed progress, streamed logs, and TOTP QR display.
- Improved OpenVPN setup and local NetworkManager import with configurable VPN CIDRs, split-tunnel defaults, aligned server settings, and hostname-based client profile names.
- Improved the remote master wizard defaults and layout with configurable parent paths, swap-size presets and custom swap sizes, optional root-password rotation, current-public-IP autofill, and removal of the CoinMarketCap API-key prompt.
- Improved VPS Manager and Services actions so systemd-managed PBGui services are started, stopped, restarted, monitored, and migrated through user units when available, with legacy fallback where needed.
- Improved remote maintenance playbooks for branch changes, Linux updates, swap, logging, Python migration, V7 config sync, API-key backups, and service restarts to respect custom install directories and systemd services.
- Made V7 config sync reconcile remote `status_v7.json` when a VPS watcher starts, pull locally missing legacy instances without per-instance timestamps, and ignore stale remote manifests for delete decisions.
- Cleaned generated Pareto preset bounds so expanded ranges use step precision or compact significant digits instead of long floating-point tails.
- Added a detailed architecture plan for future self-hosted multi-master cluster sync without external state services.
- Updated README installation, running, and service documentation for the new installer, local setup, Docker/manual alternatives, systemd services, and VPS migration.
