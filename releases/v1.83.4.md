# v1.83.4

## Highlights

- Auth: wrote refreshed API session tokens atomically to prevent brief false `401` logouts during keep-alive polling.
- VPS Manager: renamed the existing-host setup view to `Change VPS`, renamed `Save VPS` to `Apply VPS Changes`, and kept initial setup actions only for VPS entries that still need setup.
- VPS Manager: applied Swap Size changes through `Apply VPS Changes` and removed the separate `Resize Swap` action.
- VPS Manager: applied firewall setting changes automatically after `Apply VPS Changes` and removed the redundant `Update Firewall Settings` sidebar action.
- VPS Manager: prompted for the VPS user password before `Read VPS settings` or remote apply when no current field value or session secret is available.
- VPS Manager: added `Read VPS settings` progress feedback as a progress bar with task states for password check, SSH connect, remote config read, UFW read, and save stages.
- VPS Manager: fixed `Read VPS settings` firewall detection to parse real `ufw status` output via sudo even before the VPS has a `[firewall]` section in `pbgui.ini`.
- VPS Manager: fixed `Read VPS settings` so freshly entered passwords and freshly read settings are applied immediately instead of being overwritten by stale page detail.
- VPS Manager: preserved focus in Change VPS fields such as Allowed SSH IPs while live status refreshes arrive.
- VPS Manager: persisted firewall settings in VPS `pbgui.ini` for cross-master sync.
- API Sync: refreshed the VPS Manager status automatically after pulling newer API keys from another VPS.
- VPS Manager: fixed local `/etc/hosts` writes so successful additions report success and the sudo password is never passed to `tee` as file content.
- VPS Manager: fixed systemd migration preview status for optional PBRemote/CoinData services and ignored the preview probe itself when detecting legacy processes.
- VPS Manager: fixed optional PBRemote/CoinMarketCap setup edits, added targeted optional-settings apply, and guarded PBv7 `dynamic_ignore` targets without confirmed CoinMarketCap support.
