- Reduced API restart latency by closing persistent navigation status streams before shutdown, avoiding duplicate startup permission walks, yielding before deferred VPS connections, and incrementally rescanning only changed managed files and appended log bytes for migrated credential leakage.
- Unified the PBv7 and PBv8 Results browser with a version filter and mixed-version equity comparison that loads every selected result from its owning backend.
- Fixed V7-result-to-V8 conversion preserving normalized fee defaults instead of the effective linear-market maker and taker rates evidenced by the historical fills.
- Fixed PBv8 and mixed PBv7/PBv8 result deletion by keeping the action enabled in Both mode and routing every selected result to its owning backend.
- Moved Balance Calculator from PBv7 to Information, added PBv8 risk/strategy config support, enabled PBv8 Backtest handoffs and inline calculations, and linked PBv7 Run directly to the shared page.

- Hardened PB8 config persistence by publishing `backtest.json` and sparse coin-override files as one recoverable bundle, preserving PBGui metadata through PB8 canonicalization, and retaining pending browser edits until the complete save succeeds.
- Aligned PB8 updates with PB7's verified-official-ref workflow, protected new PB8 launches and API restarts with the shared update lease while leaving running backtests independent, and fixed rejected parallel commands changing the active task status.
- Fixed PB8 launch-time PBGui market-data selection, `approved_coins: all` balance calculations, authoritative final balance/equity extraction from result CSV artifacts, and credential scan-cache invalidation when the in-memory sentinel set changes.
- Fixed the PB8 Results panel accepting a transient empty response directly after an API restart by retrying bounded initial loads and ignoring stale overlapping responses.
- Hardened project agent diagnostics to prohibit raw reads of secret-bearing configuration files and require narrowly allowlisted non-secret key inspection.
